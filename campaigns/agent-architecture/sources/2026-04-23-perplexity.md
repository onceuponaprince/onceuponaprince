Here is a comprehensive, operationally-grounded answer to all six sub-questions.Agent Architecture as a Spectrum: Task-Level Routing in ProductionThe core claim holds up under current evidence: agent architecture is genuinely a spectrum, and the correct position on it is determined by task topology — parallelizability, tool-use density, state dependency — not by product identity. The routing primitive is an emerging first-class concern in the 2024–2026 production stack.1. Monolithic vs. Granular: What the Literature Says (2022–2026)The most authoritative recent data comes from a December 2025 arXiv paper, "Towards a Science of Scaling Agent Systems," which evaluated 260 configurations across six benchmarks and five canonical architectures. Its headline finding is stark: relative performance change vs. single-agent ranges from +80.8% to –70.0%, entirely depending on task type. Decomposable, parallelizable tasks (e.g., financial reasoning) benefit from multi-agent coordination; sequential planning tasks are actively harmed by it.infoq+1Google Research's controlled evaluation of 180 agent configurations, covered by InfoQ in February 2026, quantified this further: centralized multi-agent coordination produced an 80.9% improvement on parallelizable tasks, but introduced a tool-use bottleneck — as tasks required more API/tool calls, coordination costs began to outweigh the benefits. Independent agents amplified errors ~17× without a central verifier, compared to ~4.4× with centralized coordination.infoqThe O'Reilly Radar (February 2026) notes that multi-agent system papers grew from 820 in 2024 to over 2,500 in 2025, but volume hasn't resolved the fundamental question: there is no architecture that wins universally.oreilly2. Blind Task Execution: Is Context Loss a Net Reduction or Net Increase in Failure Cost?Operationally, context loss in agent systems is a net increase in failure cost — not because it always produces worse outputs, but because it makes failures opaque and non-reproducible. The Reddit practitioner consensus (r/AI_Agents, October 2025) is unambiguous: "reasoning blindness" — where the execution path is logged but the rationale behind decisions is not — is the primary debugging obstruction. The recommended fix of logging the full prompt and tool descriptions before each call "seems obvious but is often overlooked" in production systems.redditThe cost structure breaks down as:Short-context single-agent failure: fast to diagnose (one prompt, one output, clear diff)Multi-agent failure with stripped context at handoff boundaries: the error may have originated two agents upstream, the propagated output looks locally valid, and tracing requires reconstructing state that was discardedredditMulti-agent failure with error amplification: without centralized verification, the ~17× error propagation rate means a single upstream hallucination can corrupt the entire pipeline before any agent raises an exceptioninfoqMicrosoft's engineering blog (August 2025) explicitly named "single agent stretched past its breaking point" as a failure mode — but the fix they shipped (hierarchical orchestrator + domain-specialist agents) required that the orchestrator retain cross-domain context, not strip it. The insight is that context stripping should happen within agents (clean, focused prompts), not between agents (handoffs must carry enough state for verification).developer.microsoftThe Portkey observability framework tracks "audit completeness" — percentage of requests with full trace coverage — as a governance KPI precisely because missing-trace events are the highest-cost failure class.portkeyThe Routing Primitive: What It Actually Looks LikeThe current production-grade routing primitive is a three-layer cascaded controller, as formalized in MasRouter (ACL 2025):zylosCollaboration mode determiner — should this task be solved by one agent or many? (inputs: task complexity score, tool-use count estimate, sequential dependency flag)Role allocator — what roles/specializations are needed if multi-agent?LLM router — which model backbone fits each role given cost and capability constraints?In LangGraph, this maps concretely to a routing node that emits a structured output classifying the task, which then conditions the downstream graph branch. RouteLLM implements the simplest version: a BERT-class classifier trained on human preference data routing between strong/weak model pairs, achieving 85% cost reduction at 95% quality parity. The 50–200ms latency overhead of a routing classifier is only justified above ~500 agent calls/day; below that, static assignment is more efficient.augmentcode+1The routing decision is therefore not a design-time product choice but a runtime inference made per task, using a cheap classifier (Haiku-class) as the meta-planner before any expensive execution begins.breyta+1
