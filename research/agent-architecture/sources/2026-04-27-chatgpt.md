**1. Predictive Features**
1. `Verification affordance pack` [Rare]: can the route self-check cheaply via tests, schema validation, citations, diff review, or tool outputs; cheap-verifiable tasks can take cheaper agents.
2. `Coordination-tax pack` [Rare]: predicted number of subtasks, dependency depth, handoff count, likely duplicate context transfer, expected multi-agent overhead vs single-agent.
3. `Observation-deficit pack` [Rare]: how much key info is missing from the prompt; signals include unresolved pronouns, absent file refs, implicit goals, “fix this” without artifact, likely need for discovery before execution.
4. `Surface telemetry pack`: prompt length, code-vs-prose ratio, log density, stack-trace tokens, file paths, URLs, JSON/YAML blocks, shell syntax, screenshot/attachment count.
5. `Interaction-history pack`: retry count, rephrase delta from prior turn, prior route failures, prior tool-use success, context-compaction frequency, user’s tendency to ask follow-ups vs one-shot tasks.
6. `User-preference pack` [Rare]: latency tolerance, verbosity preference, acceptance of tool use, prior “don’t browse / use subagent / just answer” behavior, spend sensitivity by user/account.
7. `Temporal-state pack` [Rare]: local time-of-day, session age, user response latency, urgency markers, typo rate, weekend/night patterns; useful for “fast-enough now” vs “deep-and-slow” routing.

**2. Training Data**
1. `Self-instrumented founder traces` [Rare]: mine your own Claude Code / Agents SDK sessions, including route chosen, recovery path, tool outcomes, and whether you manually overrode the route.
2. `Public agent trace dumps` [Rare]: community Claude Code datasets, DataClaw/claudeset-style exports, public tool-use traces on Hugging Face; weak but abundant route supervision.
3. `Shadow-mode production traffic`: on a slice of real traffic, run primary + shadow route, keep both outputs, score offline; this is the highest-signal routing data source once you have users.
4. `Prediction-market labeling` [Rare]: ask raters to bet which route will win on quality/cost; use market confidence as both label and uncertainty signal.
5. `Synthetic route generation`: stronger model reads raw tasks and emits task tags, route choice, confidence, expected failure mode, and validator type; use mostly for bootstrap and edge-case coverage.
6. `Benchmark trajectory relabeling`: WebArena, OSWorld, SWE-bench, BrowseComp, GAIA-style traces can be post-labeled with “what route should have been chosen earlier.”
7. `User-paid labels / credits` [Rare]: give product credits for “was this route worth the latency/cost?” and “would you prefer monolith / graph / browse / code-exec next time?” on real tasks.

**3. Classifier Shape**
1. `Rule-prefix + learned tail`: obvious cases caught by regex/features first; ambiguous cases go to kNN or a tiny encoder.
2. `kNN over solved-task embeddings`: nearest solved tasks vote on route, cost target, and fallback; likely the strongest cheap v0.
3. `Clarify-first router` [Rare]: first class is not a route but “ask one clarifying question before routing.”
4. `Uplift router` [Rare]: predict gain of route `R` over default monolith, not absolute success; optimize incremental benefit.
5. `Simulation router` [Rare]: cheap planner predicts tool sequence / step count / validator before routing; route on the simulated plan, not raw prompt.
6. `Conformal set router` [Rare]: output top-2 safe routes or abstain on low confidence instead of forcing one label.
7. `Contextual bandit`: online updates from real outcomes; good once traffic exists and user segments diverge.

**4. Measurement Loop**
1. `Expected session regret`: compare chosen route to oracle-best route on quality, latency, and dollar cost; track regret, not just accuracy.
2. `Recovery-aware eval` [Rare]: score total session outcome after fallback/escalation, because first-hop misroutes that self-correct are less damaging than dead-end misroutes.
3. `Shadow replay`: counterfactual replay on logged tasks with alternative routes, especially on ambiguous buckets and new clusters.
4. `Route stability tests` [Rare]: paraphrase the same task, vary context length, remove one artifact, and see if routing flips too easily.
5. `Pareto frontier dashboard`: success rate, time-to-success, token cost, tool calls, and human takeover minutes saved.
6. `Golden-cluster regression`: keep a small, hand-curated set of tasks per intent cluster and block regressions cluster-by-cluster.
7. `Confidence calibration`: bucket by confidence and measure real win rate; bad calibration is worse than mediocre accuracy.

**5. Failure Modes And Fallbacks**
1. `False decomposition` -> fallback: if coordination-tax estimate is high or confidence low, collapse to monolith.
2. `Under-routing discovery tasks` -> fallback: route to scout/discovery agent first, not full executor.
3. `Catastrophic low-confidence choice` -> fallback: abstain, ask clarifying question, or require human approval for irreversible actions.
4. `Cheap-route overconfidence` -> fallback: cheapest-first, but escalate on validator fail, missing citations, failing tests, or no-file-found signals.
5. `OOD / new-domain prompts` -> fallback: freeze to safest general route and log for labeling.
6. `History contamination` -> fallback: reroute using current turn only plus compressed facts when thread drift is detected.
7. `Close-margin ambiguity` -> fallback: run top-2 routes in parallel only for high-value or high-risk tasks, then pick by validator/judge.

**6. Cost Of V0**
1. `Cheapest plausible v0`: 3-7 days, rules + kNN, 200-500 self-labeled tasks, no paid data. Cash: `$0-$300` if mostly local/open; `$50-$200` API is enough for labeling and eval.
2. `Cheap real-data v0`: 1-2 weeks, instrument your own sessions + 1k shadow-labeled tasks + offline eval harness. Cash: `$200-$1.5k`; founder time matters more than inference.
3. `Prompt-only router v0`: 2-5 days, LLM classifier with hand-written taxonomy, no training. Cash: `$50-$500`; fast to ship, brittle under drift.
4. `Month-scale robust v0`: 3-5 weeks, 5k-20k tasks, kNN or tiny encoder, synthetic augmentation, recovery policies, cluster evals. Cash: `$1k-$6k` without paid labels, `$5k-$15k` with paid labels.
5. `Quarter-scale serious solo build`: 2-3 months, contextual bandit + shadow traffic + paired judging + fallback validators. Cash: `$10k-$40k` without incentives, `$25k-$100k` if buying lots of human comparisons.
6. `Most expensive justifiable v0`: 1 quarter, 50k+ dual-route comparisons, user-segment personalization, abstention/calibration, recovery-aware evals. Cash: `$40k-$150k`; beyond this, solo-founder ROI usually breaks.
7. `Key economic point`: current API pricing makes classifier inference cheap; `gpt-5.4-mini` is `$0.75/M` input and `$4.50/M` output standard, half that in Batch, while Claude Haiku 3.5 is `$0.80/M` input and `$4/M` output. The expensive part is collecting route-outcome labels, not training the classifier.

**UNSAID OPTIONS**
1. `Verification affordance` as a first-class routing feature: route by how easy the task is to validate, not just what the task is.
2. `Coordination tax` prediction: route by whether multi-agent decomposition is worth the handoff overhead.
3. `Clarify-first` as a route label: some tasks should route to one question, not one model.
4. `Uplift modeling`: predict whether a fancy route beats the default enough to justify its cost.
5. `Prediction-market labels`: let raters bet on winning routes and use the spread as uncertainty.
6. `Recovery-aware metrics`: optimize total session success after fallback, not first-hop route accuracy.
7. `Route stability under paraphrase/context loss`: unstable routers look smart on benchmarks and fail in products.
8. `Founder trace mining from local agent logs`: probably the highest-signal overlooked bootstrap source for a solo builder.
