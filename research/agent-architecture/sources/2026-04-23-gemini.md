## Episode 1 Research Report: The Agent Architecture Spectrum

This report investigates the transition from static agent product choices to a dynamic task-level architecture spectrum, focusing on peer-reviewed literature and systems research (2022–2026).

---

### 1. Monolithic vs. Granular Task Performance (2022–2026)
The literature reveals a "pendulum swing" in architectural dominance. Early successes in granular systems (2023–2024) have been challenged by high-reasoning monolithic baselines in 2025–2026.

*   **Foundation (2022–2023):** **ReAct** (Yao et al., 2022) established the monolithic reason+act loop. **Reflexion** (Shinn et al., 2023) improved this by ~30% in ALFWorld using self-correction loops.
*   **Granular Peak (2024):** Frameworks like **MetaGPT** (Hong et al., 2024) and **ChatDev** (Qian et al., 2023) demonstrated high success in software generation by mimicking organizational SOPs (Standard Operating Procedures). MetaGPT achieved a 46.67% resolve rate on SWE-Bench Lite (Aug 2024).
*   **Monolithic Resurgence (2025–2026):** The introduction of **reasoning-token models** (OpenAI o1/o3, Claude Mythos) shifted performance back to monolithic agents. **Verdent** (2025) reported 76.1% on SWE-bench Verified using a single-loop Plan-Code-Verify pattern, while **Claude Code** (Mythos Preview, 2026) reached 93.9% on the same benchmark.
*   **Empirical Comparison:** Princeton NLP (2025) and *arXiv:2604.02460* (2026) found that single-agent systems (SAS) match or outperform multi-agent systems (MAS) on 64% of benchmarked tasks when given identical tools. MAS gains only appear when task complexity exceeds the internal reasoning depth of a single model.

### 2. Framework Exposure of the Spectrum
Frameworks are now categorized by their "granularity" rather than just their multi-agent capabilities.

*   **Fine-Grained (High Control):** **DSPy** (Khattab et al.) treats LLM calls as optimizable functions. **LangGraph** (LangChain) and **Anthropic Patterns** (2024) model agents as state machines, requiring explicit definition of every "hop."
*   **Mid-Grained (Modular):** **OpenAI Swarm** and **LlamaIndex Agents** focus on "handoffs and routines." They assume a position where the product is a collection of tools, and the agent is the router.
*   **Coarse-Grained (High Autonomy):** **CrewAI** and **AutoGen** assume a multi-agent position by default, using role-based metaphors and conversational group chats to solve emergent problems.
*   **Spectrum Choice:** Anthropic's "Workflows" (2024) explicitly argue that for 90% of tasks, a **simple router or orchestrator-worker pattern** outperforms the "autonomous agent" black box.

### 3. Context Dynamics: Improvement vs. Degradation
The performance of an agent is heavily dictated by how it interacts with its context window.

*   **Lost-in-the-Middle (Liu et al., 2023):** Established the U-shaped performance curve where information buried in the middle of a long context is ignored or degrades performance.
*   **Context Stripping Gain:** Research from *Chroma Labs* (2025) suggests that reducing "context pollution" via adversarial stripping (removing HTML/metadata) prevents model degradation. This is known as the **"Hedge Tax"**—every extra token acts as a tax on the model's attention budget.
*   **Context Degradation:** The **NoLiMa** benchmark (2025) shows that at 32k tokens, even top-tier models drop to 50% of their short-context accuracy. Sheer length acts as a "primary stressor" even when retrieval is perfect (*arXiv:2501.04112*).
*   **Load-Bearing Context:** Conversely, stripping critical constraints (e.g., specific edge-case rules in 50-page docs) causes systemic failure. The routing primitive must distinguish between "noise" and "load-bearing" context.

### 4. The Coordination-Cost Curve
Scaling agent count introduces a non-linear cost and error profile.

*   **Optimal Cluster Size:** DeepMind's *Towards a Science of Scaling Agent Systems* (2026) found that the "optimal cluster size" is **3–4 agents**. Beyond this, coordination noise outweighs gains.
*   **Error Compounding:** In decentralized MAS, error amplification can reach **17.2x** the level of a single agent (*Towards Data Science, 2025*). This is known as **"Hallucination Snowballing,"** where subsequent agents treat a previous error as "settled ground."
*   **The Sequential Penalty:** Parallelizable tasks see an 81% gain from MAS, but inherently sequential tasks suffer a **39–70% latency degradation** due to coordination overhead.

### 5. Measurement Surfaces and Axis Mapping
Modern evaluation has moved beyond simple accuracy to multi-dimensional surfaces.

*   **Surfaces:** **Planning Surface** (sub-goal decomposition), **Execution Surface** (tool accuracy), and **State Surface** (context drift/memory saturation).
*   **Metrics:** **Success-per-Dollar** (token efficiency), **Intent Drift Rate** (loss of original goal), and **Human Cognitive Drift Rate** (erosion of user expertise).
*   **Benchmarks:**
    *   **GAIA (General AI):** Real-world reasoning.
    *   **AgentBench:** System-level tasks (OS/DB/Knowledge Graph).
    *   **WebArena:** End-to-end web navigation.
    *   **SWE-bench:** High-horizon code resolution.
*   **Axis Mapping:** Plotting agents on **Reliability vs. Cost vs. Autonomy** radar charts.

### 6. Debugging Cost of Blind Task Execution
Context-stripping creates an "Observability Gap" that complicates debugging.

*   **Distributed Causality:** In granular systems, bugs are often **emergent**—not in any single agent but in the "hand-off" between them.
*   **Trajectory-Level Explainability:** Literature (e.g., *CHI '25*, *arXiv:2503.02068*) identifies a shift from static XAI to trajectory tracing.
*   **Debugging Loss:** Context-stripping removes the forensic evidence (the "why") needed for root-cause analysis. To mitigate this, **Co-Explainers** (*AGDebugger*) are used, but they add their own token/latency cost, often negating the initial stripping gain.

### Synthesis: The Routing Primitive
The "routing primitive" (the mechanism for choosing the spectrum position) is emerging as the core of agentic systems.

*   **TRouter (Task-Aware Routing):** Uses a multi-level task taxonomy (Domain → Subcategory → Difficulty) to select models/agents.
*   **Arch-Router:** Decouples **Route Selection** from **Model Assignment** using semantic matching against API contracts.
*   **MoMA (Mixture of Models and Agents):** Uses a context-aware state machine to dynamically select between a monolithic LLM or a specialized agent for every step of a workflow.

**Conclusion:** The correct position on the spectrum is a function of **Task Horizon** (how many steps?), **State Complexity** (how much load-bearing context?), and **Precision Requirement** (what is the cost of a 17x error trap?). Current 2026 literature supports that this choice must be made **per-task** via a routing primitive to maintain optimal cost-accuracy performance.
